# 개요

## 운영체제란?

운영체제(Operating System, OS)는 컴퓨터 하드웨어와 사용자 또는 응용 프로그램 사이의 인터페이스 역할을 하는 시스템 소프트웨어입니다. 
운영체제는 다음과 같은 주요 기능을 제공합니다:

- **프로세스 관리**: 프로그램 실행 및 스케줄링
- **메모리 관리**: 메모리 할당 및 가상 메모리 관리
- **파일 시스템**: 파일 및 디렉토리 관리
- **장치 관리**: 하드웨어 장치와의 통신
- **보안**: 사용자 인증 및 권한 관리

## Rust로 운영체제를 개발하는 이유

### 장점

1. **메모리 안전성**: 컴파일 타임에 메모리 오류를 방지
2. **성능**: C/C++ 수준의 성능 제공
3. **동시성**: 안전한 동시성 프로그래밍 지원
4. **현대적 언어**: 패턴 매칭, 타입 시스템 등 현대적 기능
5. **생태계**: 풍부한 크레이트(crate) 생태계

### 도전 과제

1. **no_std 환경**: 표준 라이브러리 사용 불가
2. **하드웨어 직접 제어**: 레지스터 및 메모리 직접 접근 필요
3. **인라인 어셈블리**: 일부 기능은 어셈블리 코드 필요
4. **디버깅 어려움**: 커널 디버깅은 복잡함

## 운영체제 개발의 기본 개념

### 부팅 프로세스

1. **BIOS/UEFI**: 하드웨어 초기화 및 부트로더 로드
2. **부트로더**: 커널을 메모리로 로드하고 실행
3. **커널 초기화**: 하드웨어 설정 및 시스템 초기화
4. **사용자 공간**: 응용 프로그램 실행 환경

### 메모리 레이아웃

```
+------------------+ 0xFFFFFFFF (4GB)
|   커널 공간      |
|   (Kernel Space) |
+------------------+ 0xC0000000
|                  |
|   사용자 공간    |
|   (User Space)   |
|                  |
+------------------+ 0x00000000
```

### 인터럽트

인터럽트는 하드웨어나 소프트웨어가 CPU의 주의를 끌기 위해 발생시키는 신호입니다:

- **하드웨어 인터럽트**: 키보드, 마우스, 타이머 등
- **소프트웨어 인터럽트**: 시스템 콜, 예외 처리
