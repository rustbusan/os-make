프로젝트 계획

프로젝트 목표

Rust를 사용하여 기본적인 운영체제를 처음부터 개발합니다. 단계별로 기능을 추가하여 완전한 운영체제로 발전시킵니다.

개발 단계

Phase 1: 기반 구축 (Foundation)

목표: 부팅 가능한 최소한의 커널 구현

  • [x] 개발 환경 설정
  • [ ] 부트로더 설정 (GRUB 또는 직접 작성)
  • [ ] 최소한의 커널 코드 작성
  • [ ] 화면 출력 (VGA 텍스트 모드)
  • [ ] "Hello World" 출력

예상 기간: 1-2주

Phase 2: 메모리 관리 (Memory Management)

목표: 기본적인 메모리 관리 시스템 구현

  • [ ] 페이지 테이블 설정
  • [ ] 힙 할당자 구현
  • [ ] 가상 메모리 매핑
  • [ ] 메모리 보호 메커니즘

예상 기간: 2-3주

Phase 3: 인터럽트 처리 (Interrupt Handling)

목표: 하드웨어 인터럽트 처리 시스템 구축

  • [ ] IDT (Interrupt Descriptor Table) 설정
  • [ ] 키보드 인터럽트 처리
  • [ ] 타이머 인터럽트 처리
  • [ ] 예외 처리 (페이지 폴트, GPF 등)

예상 기간: 2-3주

Phase 4: 프로세스 관리 (Process Management)

목표: 멀티태스킹 지원

  • [ ] 프로세스 구조체 정의
  • [ ] 프로세스 스케줄러 구현
  • [ ] 컨텍스트 스위칭
  • [ ] 시스템 콜 인터페이스

예상 기간: 3-4주

Phase 5: 파일 시스템 (File System)

목표: 기본적인 파일 시스템 구현

  • [ ] 파일 시스템 구조 설계
  • [ ] 디렉토리 및 파일 관리
  • [ ] 파일 읽기/쓰기 구현
  • [ ] 가상 파일 시스템 (VFS) 인터페이스

예상 기간: 3-4주

Phase 6: 장치 드라이버 (Device Drivers)

목표: 주요 하드웨어 장치 지원

  • [ ] 키보드 드라이버
  • [ ] 마우스 드라이버
  • [ ] 디스크 드라이버
  • [ ] 네트워크 드라이버 (선택사항)

예상 기간: 4-6주

기술 스택

필수 도구

  • Rust: 최신 안정 버전 (1.70+)
  • QEMU: 가상 머신 에뮬레이터
  • GDB: 디버거
  • GRUB: 부트로더 (또는 직접 작성)
  • NASM: 어셈블러 (필요시)

Rust 크레이트

  • bootloader: 부트로더 통합
  • x86_64: x86_64 아키텍처 지원
  • spin: 스핀락 구현
  • volatile: 휘발성 메모리 접근

프로젝트 구조

os-make/
├── Cargo.toml
├── Cargo.lock
├── bootloader/          # 부트로더 설정
├── src/
│   ├── main.rs         # 커널 진입점
│   ├── memory/         # 메모리 관리
│   ├── interrupts/     # 인터럽트 처리
│   ├── process/        # 프로세스 관리
│   ├── fs/             # 파일 시스템
│   └── drivers/        # 장치 드라이버
├── docs/               # 문서
└── tests/              # 테스트 코드

성공 기준

각 단계가 완료되면 다음을 확인할 수 있어야 합니다:

  1. Phase 1: QEMU에서 커널이 부팅되고 "Hello World" 출력
  2. Phase 2: 동적 메모리 할당이 정상 작동
  3. Phase 3: 키보드 입력이 처리되고 화면에 표시
  4. Phase 4: 여러 프로세스가 동시에 실행됨
  5. Phase 5: 파일을 읽고 쓸 수 있음
  6. Phase 6: 주요 하드웨어 장치가 정상 작동

리스크 관리

주요 리스크

  1. 하드웨어 호환성: 다양한 하드웨어에서 테스트 필요
  2. 디버깅 어려움: 커널 디버깅은 복잡하고 시간 소모적
  3. 문서 부족: 일부 하드웨어 문서가 불완전할 수 있음
  4. 성능 최적화: 초기 구현은 최적화되지 않을 수 있음

완화 전략

  • 단계별 개발 및 테스트
  • QEMU를 통한 가상 환경에서 먼저 개발
  • 커뮤니티 및 참고 자료 활용
  • 코드 리뷰 및 문서화
